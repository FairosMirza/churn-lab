Honest limitations

            
    
              
  • Synthetic data. The generator encodes plausible, publicly-documented
              behaviour; real data will be noisier and drivers must be re-estimated.
    • 
              
  • Counterfactuals ≠ causality. The what-if engine assumes the model's
              feature–risk relationship is causal for the mutated feature. It ranks bets; A/B tests
              confirm them.
    • 
              
  • Save rates are assumptions. Playbook impact uses planning numbers,
              clearly labelled, until pilots measure real treatment effects.
    • 
              
  • Attribution is one-at-a-time. Local explanations mutate one feature
              at a time (no interaction effects) — fast and intuitive, less exact than SHAP.
    • 
            

          

          

            
From prototype to production

            
    
              
  • Label definition: replace the simulated flag with an observed one
              (e.g. no session in 60 days), with a locked feature-cutoff date to prevent leakage.
    • 
              
  • Calibration & monitoring: calibrate probabilities (isotonic),
              track AUC/PR and feature drift per market, retrain on schedule.
    • 
              
  • Serving: batch-score nightly into the personalization layer;
              expose segments to CRM/offer systems; log outcomes back for the next retrain.
    • 
              
  • Experimentation: wire the playbook to the experiment platform —
              every action ships with its primary metric and guardrails (see What-if studio).
    • 
            

          

        

      

      

        
Stack

        
Python · pandas · scikit-learn (LogReg / Random Forest / HistGradientBoosting,
        5-fold stratified CV, permutation importance) · FastAPI backend with an in-memory run registry ·
        dependency-free HTML/CSS/JS frontend · Claude (Opus 4.8) for narration with a deterministic
        fallback · fully reproducible from a seed. Sources for every pain point are in
        RESEARCH.md.
